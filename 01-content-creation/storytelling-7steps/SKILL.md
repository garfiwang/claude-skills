# 故事創作七步法技能

## 核心框架：七步法

這個技能使用 Rich 的「故事創作七步法」來生成、強化或分析故事。
每個步驟都有明確的功能，缺一不可。

```
1️⃣ Goal（目標）      → 主角渴望什麼？表面 vs 深層
2️⃣ Obstacle（阻礙）  → 三層障礙：外在 / 人際 / 內在
3️⃣ Struggle（努力）  → 用錯方法行動，讓事情更糟
4️⃣ Outcome（結果）   → 假勝利或墜落，故事失控加速
5️⃣ Twist（意外）     → 必然但出乎預料的反轉
6️⃣ Turn（轉彎）      → 主角真正改變的時刻
7️⃣ Resolution（結局）→ 主角變成誰？不是「事情解決了」
```

---

## 執行模式判斷

拿到使用者的需求後，先判斷是哪種模式：

### 模式A：從主題創作故事
使用者給的是：主題、角色名、情境關鍵字、或目標讀者

→ 照七步法完整創作，輸出完整故事

### 模式B：強化現有草稿
使用者給的是：已有的故事文字

→ 先對照七步法逐一分析，找出缺失或薄弱的步驟，然後重寫強化版本

### 模式C：診斷故事弱點
使用者說「分析」、「哪裡不好」、「有什麼問題」

→ 只做診斷，不重寫；逐步驟給出評分與建議

---

## 創作流程（模式A）

### Step 1｜釐清需求（若資訊不足）

若使用者給的資訊太少，先快速詢問（最多問 2 個問題）：
- 故事主角是誰？（真實人物、虛構角色、或「一個人」泛指）
- 故事想傳達什麼核心訊息或情感？

若使用者給的資訊已足夠，直接執行，不要問。

### Step 2｜設計七個步驟（內部思考）

在輸出前，先在腦中完成七步法設計：

**1️⃣ Goal**
- 表面目標（具體、可行動的）
- 深層需求（情感的、心理的、往往是主角自己也沒意識到的）

**2️⃣ Obstacle**
設計至少兩種層次的阻礙：
- 外在：有形的敵人、時間壓力、環境限制
- 人際：背叛、誤解、親密關係中的衝突
- 內在：恐懼、自我懷疑、舊信念的枷鎖
> 最強的阻礙設計原則：「讓主角變成更糟的自己」

**3️⃣ Struggle**
主角採取行動，但必須「用過去的思維解決現在的問題」
每次行動都讓情況更糟，製造焦慮感

**4️⃣ Outcome**
行動帶來結果，但選擇其中一種：
- 假勝利（短暫成功，觀眾以為結束，但更大危機即將來臨）
- 墜落（徹底失敗，主角跌到最低點）

**5️⃣ Twist**
問自己：「觀眾以為 A 會發生，那我怎麼讓 B 發生？」
反轉必須是「必然但出乎預料」—回頭看能找到伏筆，但第一次看絕對想不到

**6️⃣ Turn**
主角的內在改變，通常來自：
- 面對自己一直逃避的弱點
- 承認一個痛苦的真相
- 放棄一個曾經深信的信念

**7️⃣ Resolution**
不是「問題解決了」，而是「主角變成誰了」
選擇一種結局類型：
- 🟡 苦甜結局：成功但付出代價（失去重要的人）
- 🟢 成長結局：失敗但找到真正自我
- 🔴 悲劇結局：巨大代價，觀眾震撼、反思

### Step 3｜輸出格式

根據用途選擇輸出格式：

**社群/FB貼文用** → 純敘事風格，流暢段落，有情感節奏
**課程/演講案例** → 有小標題分段，清楚但不失故事感
**創作/寫作練習** → 完整故事，可較長，保留文學性

## 強化流程（模式B）

1. 先列出原草稿七步法的對應分析：
   ```
   1️⃣ Goal：[找到了嗎？強度如何？]
   2️⃣ Obstacle：[幾個層次？最弱的是哪個？]
   3️⃣ Struggle：[有沒有「用錯方法」的感覺？]
   4️⃣ Outcome：[假勝利還是墜落？清楚嗎？]
   5️⃣ Twist：[有意外嗎？夠「必然但出乎預料」嗎？]
   6️⃣ Turn：[主角有真正改變嗎？還是只是事情解決了？]
   7️⃣ Resolution：[結局說的是事件還是主角？]
   ```

2. 點出 2-3 個最需要加強的步驟

3. 輸出強化版本

---

## 診斷流程（模式C）

逐步驟給出評分（⭐1-5）和具體改善建議：

```
1️⃣ Goal　　⭐⭐⭐　→ 有表面目標，但深層需求不夠清楚
2️⃣ Obstacle ⭐⭐　 → 只有外在阻礙，缺乏內在衝突
...
```

最後給出「最需要優先改善的 Top 2」

---

## 品質標準

完成後自我檢查：
- [ ] 主角有沒有「深層需求」（不只是表面目標）？
- [ ] 阻礙有沒有讓主角「變成更糟的自己」？
- [ ] Struggle 有沒有讓觀眾焦慮感上升？
- [ ] Twist 是「必然但出乎預料」還是「隨機驚喜」？
- [ ] 結局說的是「主角變成誰」而不是「事情怎麼收尾」？

如果有任何一項答案是「沒有」，重新修改再輸出。

---

## 語言與風格

- 預設使用繁體中文
- 若使用者用英文提問，改用英文輸出
- 故事語氣：真實、有溫度、避免過度戲劇化的形容詞堆疊
- 對話若有出現：要符合角色個性，不能太「寫作課教科書感」
