# 新聞排版技能

## 觸發情境
使用者說：「新聞排版」、「簡報新聞版」、「幫我排版這篇新聞」、「做成報紙版面」等。

## 腳本位置
`~/.claude/skills/newspaper-pptx/build.js`

---

## 執行流程

### Step 1：解析新聞內容

從使用者提供的文章中提取以下欄位，**自行判斷**，不要向使用者逐一確認：

| 欄位 | 說明 | 限制 |
|------|------|------|
| `publication` | 報社名稱 | 預設「自由時報」 |
| `date` | 日期＋版次 | 例：2023年09月03日　星期日　社會版 |
| `reporter` | 記者行 | 例：記者吳仁捷／台北報導 |
| `headline_line1` | 主標題第一行 | 不超過 12 字 |
| `headline_line2` | 主標題第二行 | 不超過 14 字（可為空） |
| `subhead_line1` | 副標題第一行 | 不超過 32 字 |
| `subhead_line2` | 副標題第二行 | 不超過 32 字（可為空） |
| `col1` | 第一欄內文 | 150–250 字，段落開頭加「　　」 |
| `col2` | 第二欄內文 | 150–250 字，段落開頭加「　　」 |
| `facts_title` | 右側資訊框標題 | 例：案例關鍵資訊 |
| `facts` | 資訊框內容 | 陣列 [["標籤","值"], …]，最多 7 筆，標籤≤4字 |
| `legal_note` | 底部紅色提醒 | 一行，不超過 30 字（可為空） |
| `output` | 輸出檔名 | 預設 newspaper_output.pptx，存在當前目錄 |

**內文欄位原則：**
- col1 放文章前半（案例背景、事件經過）
- col2 放文章後半（後果、反應、追蹤）
- facts 提取關鍵數字、人物、時間、法律後果，濃縮成條列
- legal_note 放最重要的法律依據或核心啟示（一句話）

### Step 2：寫出 JSON 設定檔

將提取的內容**寫入** `/tmp/newspaper_config.json`，格式如下：

```json
{
  "publication": "自由時報",
  "date": "2023年09月03日　星期日　社會版",
  "reporter": "記者吳仁捷／台北報導",
  "headline_line1": "嬤遺囑僅蓋章未簽名　",
  "headline_line2": "家屬控律師讓3千萬飛了",
  "subhead_line1": "代筆律師與見證律師均未親筆簽名　國稅局認定遺囑無效　訴願遭駁回",
  "subhead_line2": "長子決定提告三位律師求償三千萬元",
  "col1": "　　新北市八十二歲陳姓老婦人…",
  "col2": "　　黃男指控，代筆遺囑與見證的律師團…",
  "facts_title": "案例關鍵資訊",
  "facts": [
    ["遺囑人", "82歲陳婦（新北市）"],
    ["祖產", "透天厝，市值逾 3,000 萬元"],
    ["預立遺囑", "2022 年 12 月，立後 8 天辭世"],
    ["致命錯誤", "均蓋章，未親自簽名"],
    ["法律效果", "遺囑無效（民法 §1194）"],
    ["繼承後果", "四子女法定繼承，長子僅剩 1/4 特留分"],
    ["提告求償", "長子提告三律師，求償 3,000 萬"]
  ],
  "legal_note": "民法§1194：代筆遺囑須「親自簽名」，蓋章無效",
  "output": "newspaper_output.pptx"
}
```

### Step 3：執行腳本

```bash
node ~/.claude/skills/newspaper-pptx/build.js /tmp/newspaper_config.json
```

### Step 4：回報

告知使用者輸出檔案的完整路徑。

## 版面預覽

```
┌─────────────────────────────────────────────────────────────┐
│ [報社名] (紅字)                      [日期 版次] (灰字)     │
├─────────────────────────────────────────────────────────────┤
│  主標題第一行（34pt 粗體）                                   │
│  主標題第二行（34pt 粗體）                                   │
│  副標題第一行（14pt 粗體）                                   │
│  副標題第二行（14pt 粗體）                                   │
├─────────────────────────────────────────────────────────────┤
│▌ [記者行]                   │ ┌──────────────────────────┐ │
│  第一欄內文 10pt            │ │ [facts_title] (藍底白字) │ │
│                             │ ├──────────┬───────────────┤ │
│  第二欄內文 10pt            │ │ 項目     │ 內容          │ │
│                             │ ├──────────┼───────────────┤ │
│                             │ │ 標籤     │ 值 (交替底色) │ │
│                             │ └──────────┴───────────────┘ │
│                             │ [legal_note] (粉紅底紅字)     │
└─────────────────────────────┴──────────────────────────────┘
```

## 設計規範
- 字型：微軟正黑體（全頁統一）
- 頂部：報社名（紅字 #CC0000）與日期（灰字 #555555），下方為粗黑線分隔。
- 主標題：34pt 粗體 ／ 副標題：14pt 粗體 ／ 內文：10pt
- 背景：純白 ／ 主文字：黑色（#111111）
- 第一欄左側紅色邊條，不加垂直欄間線。
- 資訊框：表格標題列藍色（#548CA8，白字），交替淺藍灰底色（#F2F6F8）、細灰格線，底端灰色來源與製表欄。
- 頁面底端黑色細線。
