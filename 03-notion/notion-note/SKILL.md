# 加入 Notion Note 資料庫

使用者想把內容存進 Notion 筆記資料庫。你的任務是：分析內容 → 決定欄位 → 呼叫工具新增頁面 → 回報結果。

---

## 目標資料庫

- **data_source_id**：`201c9c67-1f57-810a-8466-000b862f50e1`
- **MCP 工具**：`mcp__b06b418b-2b7b-414f-8817-1201b8e09cfe__notion-create-pages`

## 欄位決策規則

### 1. `Note`（頁面標題，必填）
根據內容幫使用者命名，原則：
- 簡潔有力，15字以內
- 若是文章：取核心主題（例：「貝氏更新 × 聯準會通膨誤判」）
- 若是靈感/想法：用一句話抓住精髓
- 若是待辦：以動詞開頭（例：「整理 BNI 名單」）
- 若使用者已給標題：直接沿用

### 2. `Type`（擇一）
| 選項 | 適用情境 |
|---|---|
| `Thoughts` | 個人想法、靈感、觀點、心得 |
| `Notes` | 筆記、文章摘要、學習紀錄 |
| `Research` | 研究、數據、報告、新聞分析 |
| `Documentation` | 操作說明、流程、參考資料 |

### 3. `Tags`（多選，JSON array，從下列選 1-3 個最相關）
```
AI、新聞、好文章、利率財經、信託、退休、投資風險、財富管理、
自媒體、chatgpt、靈感、增員、繼承、上課筆記、美元、美債、
詐騙、財富管理、傳承規劃、自我成長、金句、小故事、
好書、讀書心得、稅務法規、遺產稅、民法、意定監護、失智、
品牌行銷、行銷技法、BNI、100天發文、threads、tiktok
```
若都不符合，可留空陣列 `[]`。

### 4. `分類`（擇一）
- `靈感`：創意、點子、突發想法
- `資訊`：新聞、知識、參考資料
- `待辦`：需要行動的事項
- `笑話`：幽默內容
- `其他`：無法歸類

### 5. `標籤`（多選，JSON array，從下列選 0-2 個）
`設計`、`寫作`、`商業`、`生活`、`技術`、`演講`

### 6. `內容`（text，必填）
100 字以內的中文摘要，說明這則筆記的核心內容。

### 7. `來源`（text，選填）
若內容有明確來源（網址、書名、演講者、媒體名稱），填入。
若無，留空字串 `""`。

## 執行步驟

1. **讀取使用者輸入**：接收他們貼上或描述的全部內容
2. **決定所有欄位**：依上方規則推斷，不要問使用者（除非內容完全看不懂）
3. **呼叫 notion-create-pages**：
   ```
   parent: { type: "data_source_id", data_source_id: "201c9c67-1f57-810a-8466-000b862f50e1" }
   pages: [{
     icon: 適合內容的 emoji（例：📝 / 💡 / 📰 / 📚）,
     properties: {
       Note: "頁面標題",
       Type: "Notes",
       Tags: "[\"標籤1\", \"標籤2\"]",
       分類: "資訊",
       標籤: "[\"商業\"]",
       內容: "摘要文字",
       來源: "來源或空字串"
     },
     content: 完整內容（Markdown 格式）
   }]
   ```
4. **回報結果**：
   ```
   ✅ 已新增到 Notion！
   📄 頁面標題：[標題]
   🔗 [開啟頁面](notion頁面URL)
   ```

---

## 注意事項

- **不要詢問欄位**：直接判斷，只有在內容完全無法辨識時才問
- **內容完整保留**：page body 要放入使用者提供的完整內容，不要縮減
- **一次只建一個頁面**：除非使用者明確要求多頁
- **Tags 用 JSON array 字串格式**：`"[\"AI\", \"新聞\"]"`（Notion MCP 的要求）
